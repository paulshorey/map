Good question, and the answer is clean: **the map libraries do not do this.** Leaflet, MapLibre, OpenLayers, etc. only _render_ tiles and markers — they have no built-in database of place names. Turning "small town, state, country" into coordinates is a separate service called **geocoding** (specifically _forward geocoding_, and the autocomplete-as-you-type variant is _search/autocomplete_). So yes, you need a geocoding API — but the good news is the same OpenStreetMap data behind your free basemap also powers comprehensive, cheap geocoders.

What you're describing is **geocoding** (text → coordinates), and yes, you need a separate geocoding service for it.

The good news: the same **OpenStreetMap data** behind your free basemap is also the most comprehensive free place database in the world. It covers small unknown towns, plus full administrative hierarchies — countries, states/provinces, counties, regions, cities, suburbs, postal codes, even POIs. And it's built for exactly your disambiguation case: an ambiguous query like "Springfield" returns **multiple ranked candidates**, each with a `display_name` (e.g. "Springfield, Illinois" vs "Springfield, Missouri") and a **bounding box** you can feed straight into `map.fitBounds()` to jump there. So you don't have to build the "narrow it down" list yourself — the geocoder hands it to you.

**Important caveat on the public Nominatim server:** OpenStreetMap's usage policy caps it at an absolute maximum of **1 request per second**, requires a valid identifying User-Agent and attribution, prohibits heavy/systematic use, and explicitly **discourages serious business use**. Critically for you, this makes it **unsuitable for type-ahead autocomplete** — every keystroke is a request, so a single search box would blow past the limit instantly and risk getting blocked. Please review the full policy before relying on it: https://operations.osmfoundation.org/policies/nominatim/. For anything beyond occasional prototype lookups, use a hosted provider (LocationIQ et al.) or self-host.

**My recommendation for your stack:** start with **LocationIQ's free tier** for search + autocomplete — its API mirrors Nominatim's, so if you later self-host Nominatim alongside your tiles, migrating is nearly a drop-in swap. That keeps you at $0 now and gives you a free, unlimited self-hosted path later, all on the same OSM data you're already using for the map.
